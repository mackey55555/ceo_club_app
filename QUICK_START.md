## 動作確認の手順

### 1. データベース確認 ✅
既に完了しています：
- ✅ マイグレーション実行済み
- ✅ シードデータ投入済み

### 2. モバイルアプリの起動

```bash
cd mobile-app
npm start
```

**確認ポイント：**
- Expo開発サーバーが起動する
- QRコードが表示される
- ログイン画面が表示される（エラーがないこと）

**トラブルシューティング：**
- エラーが出る場合は、`npm install` を再実行
- Expo Goアプリをスマートフォンにインストール
- QRコードをスキャンしてアプリを開く

### 3. 管理画面の起動

```bash
cd admin
npm run dev
```

ブラウザで http://localhost:3000 にアクセス

**確認ポイント：**
- `/admin/login` にリダイレクトされる
- ログイン画面が表示される
- エラーが表示されない

### 4. Web（非会員申込ページ）の起動

```bash
cd web
npm run dev
```

ブラウザで http://localhost:3001 にアクセス

**確認ポイント：**
- エラーなく表示される

## 重要な注意事項

### Supabaseキーの形式について

提供されたキーが `sb_publishable_` で始まっている場合、新しいSupabase形式の可能性があります。

**通常の形式：** `eyJ...` で始まるJWTトークン

もし接続エラーが発生する場合は：
1. Supabaseダッシュボードの「Settings」→「API」を開く
2. 「Project API keys」セクションで「anon/public」キーを確認
3. 正しいキーを環境変数に設定
